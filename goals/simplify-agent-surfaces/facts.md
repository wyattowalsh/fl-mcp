# Facts

- The root FastMCP server, when constructed for normal/default agent use, exposes only a small public surface of 8-15 high-level workflow-oriented tools, first-class @mcp.prompt entries, and curated resources/templates. The exact composition and style follows the latest advanced, community-driven best practices for FastMCP v3 and MCP usage (as of 2026-06-06 and forward).
- The complete FL power surface (FL-Studio-API-Stubs typed primitives for channels/mixer/playlist/transport/UI/patterns/devices/arrangement/etc., full bridge operations, and optional Flapi-style deep remote exec/eval) is fully implemented, typed, and usable for maximum control, but is organized behind explicit tags (fl, internal, automation, power) and/or mount namespaces (e.g. fl.*) or dual-mode construction so that it is never present in the default public agent view. The design principle is 'whatever simplifies the agent experience in utilizing FL Studio the most while retaining the most control/usability'.
- Public surface curation is modeled on ableton-mcp patterns (curated high-level operations, excellent task-oriented descriptions, domain grouping, remote-script/MCP separation lessons). Power surface contracts and naming are informed by official FL-Studio-API-Stubs (canonical typed surface). Optional deep power supports Flapi patterns (MIDI sysex remote exec/eval, client/server contexts, stdout capture, REPL-style). All of the above are grounded in official FastMCP v3 patterns (mount, tags+enable/disable, providers, transforms, first-class prompts + Context, ToolResult, automatic schemas, Client testing, dual-mode) and MCP progressive discovery / large-catalog ergonomics guidance. Any other authoritative sources current as of 2026-06-06 are incorporated during self-improvement cycles.
- A documented, executable self-improvement process keeps the public surface (and this goals package) recursively aligned: at every release and on a defined cadence the team (1) refreshes from official FastMCP/MCP llms.txt + llms-full.txt and key community sources, (2) repacks representative examples (ableton-mcp and other FL MCP experiments), (3) re-evaluates composition, prompt, tag, provider, and transform patterns, (4) runs plannotator (or equivalent structured review) on the public surface docs and inventory, and (5) updates the public API inventory, BUILD_STATUS.md, and this goals package. The goals package itself is treated as the living definition of the desired agent-touching interface qualities.
- Any change that adds, modifies, removes, or reorganizes items in the public agent surface (tools, prompts, or resources) is treated as a public API / agent-facing contract change. A proper OpenSpec change record is created/started specifically for the agent surface overhaul, and the goals package references the OpenSpec change ID. Machine-readable OpenSpec state is captured (via wagents openspec ... --format json or equivalent) where it adds value to the artifacts.
- All public surface changes trigger the docs-steward workflow (if available) and corresponding updates to BUILD_STATUS.md and the public API inventory.
- Representative end-to-end FL Studio workflows (e.g. apply preset to selected track, create automation lane for a parameter, obtain focused project state summary and perform a safe targeted mutation, etc.) can be completed successfully by agents using only the public surface plus prompts, without any need to reach the power surface.
- When an in-memory FastMCP Client is created against the default-constructed root server (the one normal agents receive), it sees only the small public surface. All public elements have high-quality task-oriented descriptions, appropriate annotations/tags/safety hints, and benefit from FastMCP automatic structured output.
- Existing bridge request/response contracts, live FL execution semantics (when the repo-owned bridge is used), authentication/authorization, transport behavior, and all non-surface functionality are not regressed by the overhaul. This is verified via contract tests and live FL smoke tests.
- The surface simplification, discovery, and public API sections of the prior `finish-v0-1-0-fl-surface-fastmcp` goal package (including its plannotator-approved plan and facts) are superseded and replaced by this goal package and the resulting implementation/plan.
- The goals package at goals/simplify-agent-surfaces/ (goal.md, facts.md, plan.md, and supporting JSON) is the authoritative, living definition of the desired qualities for all agent-touching interfaces in the project and is kept in sync with the implemented surface, OpenSpec changes, and BUILD_STATUS.md.
