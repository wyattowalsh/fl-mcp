# Manual FL Studio Audit

Audit the current `fl-mcp` local user experience end to end against the real local FL Studio app, with MCP-driven agent production flows as the primary product behavior. The audit must exercise or explicitly classify the compact MCP surface, resources/templates/prompts, all 196 internal FL Studio catalog operations, CLI setup and diagnostics, live bridge paths, selected-controller compatibility, helper diagnostics, docs-guided setup, and negative/error paths.

Use `facts.md` as the shared understanding of required outcomes.

Use `plan.md` as the execution plan for the audit, issue ledger, and refinement/fixer plan.

Done means all accepted facts are satisfied or have documented blockers, every catalog operation has an evidence-backed live classification, every visible flow/option has been exercised or classified, and `fixer-plan.md` contains a severity-led plan with reproduction evidence, affected systems, proposed fixes, automated regression checks, and manual FL Studio re-test gates.
