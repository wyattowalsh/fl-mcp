# FL MCP Docs

Fumadocs + Next.js documentation app.

## Commands

Run docs commands from the repository root so pnpm uses `docs/package.json` and
`docs/pnpm-lock.yaml` without climbing into parent workspaces:

- `pnpm --dir docs --ignore-workspace install --frozen-lockfile`
- `pnpm --dir docs --ignore-workspace docs:generate-reference`
- `pnpm --dir docs --ignore-workspace docs:verify-reference`
- `pnpm --dir docs --ignore-workspace lint`
- `pnpm --dir docs --ignore-workspace check`
- `pnpm --dir docs --ignore-workspace build` (static export emitted under `docs/out/`)
- `pnpm --dir docs --ignore-workspace dev`

If your current directory is already `docs/`, omit `--dir docs` and keep
`--ignore-workspace`.

## Generated Files

- `content/docs/reference/generated/*.mdx` is generated by `scripts/generate-reference.mjs`.
- `.source/`, `.next/`, `out/`, and `tsconfig.tsbuildinfo` are build outputs.
- `pnpm-lock.yaml` is docs-local and should stay next to this `package.json`.

## Content Map

- `content/docs/operation-workflows.mdx` documents compact-surface recipes.
- `content/docs/validation-runbook.mdx` documents validation command groups.
- `content/docs/live-fl-studio-smoke.mdx` documents actual-app smoke evidence.
- `content/docs/architecture/public-api-inventory.mdx` is the public MCP contract.

## Release Deploy Path

- Docs deploy workflow: `.github/workflows/docs-deploy.yml`
- Triggered by stable release tags and `workflow_dispatch`.
- Build pipeline verifies generated references, builds static site, and publishes `docs/out` to GitHub Pages.
