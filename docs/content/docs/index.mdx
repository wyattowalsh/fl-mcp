FL MCP is a local-first FastMCP server for FL Studio. The public MCP surface is
12 visible tools backed by 216 typed internal operations across 16 FL Studio
domains, 5 resources, 3 resource templates, 8 workflow prompts, and 4
providers.

> [!IMPORTANT]
> **Mock vs live coverage**
>
> - **Mock catalog (216 ops):** deterministic rehearsal and CI validation with
>   explicit `provider="mock"`.
> - **Verified live (8 ops):** smoke-safe operations confirmed on the bundled
>   host-file bridge.
> - **Attemptable live (208 ops):** forced-live catalog operations that return
>   structured errors when FL Studio does not expose a compatible callable.
>
> Live mode does not silently fall back to mock. See
> [Providers](/docs/providers) and
> [Tier B Release Runbook](/docs/validation/tier-b-release).

## Install Path

Use `uvx` for normal client usage:

```bash
uvx fl-mcp --version
uvx fl-mcp server run --mode stdio --dry-run
uvx fl-mcp doctor --format json
```

Before publication, test this checkout with:

```bash
uvx --from /absolute/path/to/fl-mcp --with-editable /absolute/path/to/fl-mcp fl-mcp --version
```

## Agent Control Loop

1. Start with `fl_status` or `fl_snapshot`.
2. Search intent with `fl_search_capabilities`.
3. Fetch schemas with `fl_get_capability_schema`.
4. Execute with `fl_execute`, `fl_batch_execute`, `fl_plan`, or `fl_apply`.
5. Verify with readback, resources, task state, or another snapshot.

Primitive FL operations are intentionally hidden from `list_tools`; use
canonical operation ids such as `mixer.set_track_volume` through the compact
executor tools.
