# Reuse Audit

Use this audit to record third-party material, patterns, or code that influence architecture and implementation.

## Audit Record Template

### Source

- Origin (project, repository, paper, standard, internal package)
- URL or canonical reference
- Version / commit / publication date

### License

- Declared license
- Obligations (notice, attribution, copyleft, redistribution constraints)
- License compatibility notes with this project

### Reuse Scope

- What is reused (code, schema, docs language, architecture pattern, algorithm)
- Boundaries of reuse (files/modules/features)
- Intended duration (temporary reference vs long-term dependency)

### Modifications

- What was modified from source
- Why modifications were required
- Maintenance implications of divergence

### Attribution

- Required attribution text
- Where attribution is placed (headers, NOTICE, docs, acknowledgements)
- Verification status (complete / pending)

### Compatibility Decision

- Decision: **Accept / Accept with constraints / Reject**
- Rationale for decision
- Follow-up actions and owners
