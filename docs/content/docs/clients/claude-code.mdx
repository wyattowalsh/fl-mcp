## Prerequisites

- `uv` is installed.
- Claude Code is configured to load local MCP server config files.
- Live FL Studio mode is optional; mock mode works without FL Studio.

## Configuration

Use FL MCP as a stdio server.

Published package:

```json
{
  "command": "uvx",
  "args": ["fl-mcp", "server", "run", "--mode", "stdio"]
}
```

Local checkout:

```json
{
  "command": "uvx",
  "args": ["--from", "/absolute/path/to/fl-mcp", "--with-editable", "/absolute/path/to/fl-mcp", "fl-mcp", "server", "run", "--mode", "stdio"]
}
```

For live FL Studio mode, run `fl-mcp install --dry-run` and copy the matching
`uvx_environment` values into the Claude Code MCP server environment. Use
`uvx_harness_environment` when validating the live subprocess boundary without
FL Studio.

## Verification

- Claude Code lists exactly 12 FL MCP tools.
- `fl_status` reports `status="ok"` and `internal_operation_count=216`.
- `fl_search_capabilities` and `fl_get_capability_schema` resolve a target
  operation.
- A representative `fl_execute` mock call succeeds before enabling live mode.
