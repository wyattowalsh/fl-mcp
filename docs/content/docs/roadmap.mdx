## Purpose

Track planned FL MCP documentation and platform delivery priorities.

## Prerequisites

- Current status from `BUILD_STATUS.md`.
- Architecture decisions recorded in ADRs.
- Public API contract status reviewed.

## Steps

1. Capture near-term milestones with explicit success criteria.
2. Record dependency order and cross-team constraints.
3. Separate planned, in-progress, and completed work clearly.
4. Link roadmap entries to ADRs, release notes, and implementation artifacts.

## Verification

- Every active roadmap item has an owner and status.
- Dependencies and sequencing are explicit.
- Outdated items are removed or marked superseded.
