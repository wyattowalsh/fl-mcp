## 2026-06-16 - FL MCP Production Flow Agent Skill

- Added the repo-local `fl-mcp-production-flow` agent skill for setup-first FL
  Studio production workflows through the compact FL MCP surface.
- Added a deterministic `setup-check.py` helper that reports JSON readiness for
  mock and live workflows without starting a persistent MCP server or mutating
  FL Studio.
- Added skill references and evals for setup gating, mock rehearsal, live-mode
  safety, render/audio evidence, and negative routing away from runtime,
  database, data-pipeline, and visual UI changes.
- Kept the public FastMCP server surface unchanged; the skill guides agents but
  does not add visible MCP tools or runtime providers.

## 2026-06-15 - Plugin Profile Extensibility

- Added priority-scoped plugin support tiers: P0 paid installed targets, P1
  paid/pro suite entries, P2 popular/useful stock or free tools, and P3
  inventory-only/on-demand rows.
- Added internal plugin-profile operations behind the existing compact 12-tool
  surface for plugin inventory, profile lookup, live probing, calibration,
  raw parameter-name enumeration, raw profile generation, priority support
  audits, semantic mapped-parameter reads/writes, local preset discovery, and
  inventory reconciliation.
- Kept plugin support profile-first: vendor/manual data seeds semantic controls,
  while executable FL parameter indices require live calibration or verified
  local overlays.
- Added seed profiles for Sylenth1, Serum/Serum2, FabFilter Pro-Q 3, FabFilter
  Pro-C 2, ShaperBox 2, detected FabFilter suite stubs, CamelCrusher, Drumazon,
  ValhallaRoom, FL Studio VST/AU handling, and an iZotope Trash desired target.
- `fl_browser`, `fl_search_capabilities`, and `fl_get_capability_schema` now
  surface plugin profile and local inventory status without adding public MCP
  tools. Uncalibrated mapped writes fail closed with structured errors such as
  `calibration_required`.
- The compact catalog now contains 216 internal operations across 16 domains;
  visible FastMCP tools remain exactly 12.

## 2026-06-13 - Forced-Live Provider Contract

- Updated `flapi-live` from a narrow advertised live subset to a forced-live
  host-file bridge route for every compact catalog operation when the runtime is
  in live mode.
- `provider="auto"` now resolves to `flapi-live` for every operation in live
  mode. Explicit `provider="mock"` remains deterministic for rehearsal and CI.
- Capability metadata reports verified bridge operations as `available` and
  best-effort live attempts as `attemptable`; neither path may report success
  unless the DAW bridge confirms the call.
- Live failures are structured as `api_missing`, `unsupported_host_behavior`,
  `path_unavailable`, `host_exception`, bridge timeout, or bridge process errors
  instead of silently falling back to mock.
- Regenerated the generated API inventory from the compact public MCP surface so
  docs verification checks the visible 12-tool roster, resources, prompts,
  providers, and 216-operation internal catalog.

## 2026-06-12 - Compact FastMCP Agent Surface

- Replaced the previous default visible 204-tool shape with one compact FastMCP surface containing exactly 12 tools:
  `fl_status`, `fl_snapshot`, `fl_search_capabilities`, `fl_get_capability_schema`,
  `fl_execute`, `fl_batch_execute`, `fl_plan`, `fl_apply`, `fl_render`,
  `fl_analyze_audio`, `fl_manage_providers`, and `fl_browser`.
- Kept the full 196-operation FL Studio catalog across 16 domains, but moved those operations behind canonical operation ids such as `mixer.set_track_volume`.
- Added capability-first migration guidance: search capabilities, fetch schema, execute or batch by operation id, then verify with readback or snapshots.
- Removed the public/full surface flag and minimal fallback path from the supported server contract; FastMCP is now required.
- Registered workflow prompts for beat building, browser/plugin loading, automation, arrangement, safe mixing, and render/analyze flows.
- Compatibility note: clients that called primitive visible tools must migrate to `fl_execute(operation_id=...)` or `fl_batch_execute`.
- Related docs: [Public API Inventory](/docs/architecture/public-api-inventory).

## 2026-06-06 - Packaged FL Host Bridge and FastMCP Task Alignment

- Added a repo-owned FL Studio host bridge: `python -m fl_mcp.bridge.host_client`
  on the MCP side plus `fl-bundle/controller/device_FL_MCP_Bridge.py` inside the
  FL Studio MIDI scripting host, so actual-app live mode no longer depends on an
  undocumented external executable placeholder.
- Kept the direct packaged bridge runner (`python -m fl_mcp.bridge.runner`) and
  `fl-bundle/controller/fl_mcp_bridge_runner.py` wrapper for harness and
  diagnostic use.
- Updated `fl-mcp install --dry-run` to report live and harness bridge commands,
  the FL host request directory, and the controller script source/target paths.
  `fl-mcp doctor` now reports the host-script setup step and executes a
  read/mutation live-harness smoke path through the subprocess bridge contract.
- Added a selected-controller subprocess adapter for sessions that already have
  `FL Studio MCP Controller` selected, allowing actual-app smoke diagnostics to
  run through `FLStudioBridge(mode="live")` and the typed bridge contracts.
- Locked the runtime dependency to `fastmcp[tasks]>=3.4.2` so native FastMCP
  task registration is available for render/export and audio analysis tools.
- Compatibility note: bridge subprocess responses must now satisfy the typed
  `BridgeLiveResponse` contract, including a required `message` field.
- Compatibility note: transaction reports can now return `validation_failed`
  for invalid known-operation payloads before bridge execution.
- Compatibility note: install and doctor output now resolve the bundled FL
  Studio controller script from the package/source tree, use the selected
  script's private `bridge/` directory as the default host bridge directory,
  and enforce private current-user ownership checks for the file bridge.
- Compatibility note: `flapi-live` provider metadata advertises the shipped
  default host-file live operations and selected-controller compatibility
  capabilities for transport playback/tempo/loop/position, mixer reads and
  reversible mixer edits, channel rack reads/routing/step/note edits, and plugin
  inspection plus indexed parameter/preset actions. The selected-controller
  tempo path uses FL's `mixer.getCurrentTempo` / `mixer.setCurrentTempo`
  functions and normalizes the observed 1000x mixer tempo scale to normal BPM
  values.
  Other FL Studio tool families remain public typed catalog entries and
  deterministic mock operations unless a provider manifest lists the exact live
  capability.
- Compatibility note: selected-controller compatibility calls are serialized by
  a lock file and return `selected_controller_busy` when the fixed-file protocol
  is already in use.
- Related docs: [Bridge Protocol](/docs/architecture/bridge-protocol).

## 2026-03-29 - Public surface semantics hardening

- Kept `query_project` read-only/idempotent by making caller-supplied graph overrides ephemeral
  instead of mutating server runtime state, while preserving both `query_project(domain, graph)`
  and legacy `query_project(graph, domain)` invocation shapes.
- Kept `manage_providers` on the public MCP surface while disabling the dynamic-loading
  `discover` and `load_module` actions by default.
- Preserved the existing public `list`, `startup`, and `shutdown` behavior to minimize
  compatibility impact for runtime inspection and lifecycle orchestration.
- Compatibility note: public calls to `manage_providers(action="discover")` or
  `manage_providers(action="load_module")` now return deterministic error responses instead of
  triggering provider discovery or module loading.
- Related docs: [Public API Inventory](/docs/architecture/public-api-inventory).
