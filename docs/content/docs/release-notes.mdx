## Purpose

Provide a chronological record of docs-visible behavior and tooling changes.

## Prerequisites

- Merged change set with validated docs checks.
- Clear summary of user-facing impact.
- Linked sources (PR, ADR, inventory updates) ready.

## Steps

1. Add a new dated entry at the top.
2. Summarize what changed and why it matters.
3. Note compatibility or migration implications.
4. Link to relevant architecture/reference docs.

## Verification

- Entry includes date, scope, and impact.
- Compatibility notes are present for contract changes.
- Linked docs targets resolve.
